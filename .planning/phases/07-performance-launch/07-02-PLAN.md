  <action>
Create inline SVG favicon and validate HTML structure for production:

**SVG Favicon Generation:**
1. Design a minimal SVG favicon representing Open Buro (stylized "O" or network node motif)
2. Keep SVG simple: < 1 KB, single color or 2-color max
3. Example structure:
   ```svg
   <svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 32 32">
     <circle cx="16" cy="16" r="14" fill="none" stroke="#3b82f6" stroke-width="2"/>
     <circle cx="16" cy="8" r="2" fill="#3b82f6"/>
     <circle cx="16" cy="24" r="2" fill="#3b82f6"/>
     <circle cx="8" cy="16" r="2" fill="#3b82f6"/>
     <circle cx="24" cy="16" r="2" fill="#3b82f6"/>
   </svg>
   ```
4. Encode SVG as data URI and add to <head>:
   ```html
   <link rel="icon" type="image/svg+xml" href="data:image/svg+xml,<svg xmlns='http://www.w3.org/2000/svg' viewBox='0 0 32 32'>...</svg>">
   ```
5. URL-encode special characters in data URI (%, #, etc.)

**SEO Meta Tag Completion:**
6. Verify all required meta tags present in <head>:
   - `<title>` (50-60 chars)
   - `<meta name="description" content="...">` (150-160 chars)
   - `<meta name="keywords" content="...">`
   - `<meta property="og:title" content="...">`
   - `<meta property="og:description" content="...">`
   - `<meta property="og:type" content="website">`
   - `<meta property="og:url" content="https://openburo.eu">`
   - `<meta property="og:image" content="https://openburo.eu/og-image.jpg">` (placeholder URL)
   - `<meta name="twitter:card" content="summary_large_image">`
   - `<meta name="twitter:title" content="...">`
   - `<meta name="twitter:description" content="...">`
   - `<link rel="canonical" href="https://openburo.eu">`
   - `<html lang="en">`

**HTML Validation:**
7. Run W3C HTML validator: `npx html-validate index.html`
8. Fix any errors (warnings acceptable if minor)
9. Verify proper heading hierarchy (h1 → h2 → h3, no skipped levels)
10. Check all required attributes present (alt on images, href on links, etc.)

**GDPR Compliance Check:**
11. Scan HTML for any third-party tracking scripts (Google Analytics, Facebook Pixel, etc.)
12. Verify no cookie-setting scripts present
13. Ensure no external tracking domains in <head> or <body>
14. Confirm: PERF-03 requirement (no third-party cookies/tracking)

**What to avoid:**
- Do NOT use external favicon file (must be inline per single-file requirement)
- Do NOT use raster favicons (PNG/ICO) - SVG scales better and is smaller
- Do NOT include tracking scripts (violates PERF-03 and GDPR)
- Do NOT use auto-generated meta descriptions (must be hand-crafted for SEO)

**Step 1 - Local File Test (PERF-03 requirement):**
1. Open Finder, navigate to `/Users/mmaudet/work/openburo/index.html`

3. Verify: Page displays correctly, all styles applied, no broken resources
4. Expected: Full page renders with dark mode, animations work, no console errors

**Step 2 - 4G Connection Test (PERF-01 requirement):**
1. Open Chrome DevTools: `Cmd+Option+I`
2. Go to Network tab → Throttling dropdown → Select "Fast 4G"
3. Hard refresh: `Cmd+Shift+R`
4. Check Network tab "Load" time at bottom: **Must be < 2 seconds**
5. Check total page weight: **Should be < 500 KB**

**Step 3 - Lighthouse Audit (PERF-02 requirement):**
1. Open Chrome DevTools: `Cmd+Option+I`
2. Go to Lighthouse tab
3. Select: ✓ Performance, ✓ Accessibility, ✓ SEO (uncheck PWA and Best Practices)
4. Click "Analyze page load"
5. Wait for audit to complete (30-60 seconds)
6. **Verify scores:**
   - Performance: **≥ 90** (green)
   - Accessibility: **≥ 90** (green)
   - SEO: **≥ 90** (green)
7. If any score < 90: Review "Opportunities" and "Diagnostics" sections, note specific issues

**Step 4 - Visual Verification:**
1. Scroll through entire page from hero to footer
2. Check: All sections animate on scroll, no layout shifts, smooth performance
3. Test mobile view: DevTools → Toggle device toolbar → iPhone 14 Pro
4. Check: Mobile hamburger menu works, timeline renders vertically, touch targets ≥ 48px
5. Test keyboard navigation: Tab through all interactive elements, verify focus visible

**Step 5 - No Tracking Verification (PERF-03 requirement):**
1. DevTools → Network tab
2. Hard refresh page
3. Filter by "All" → scan request domains
4. **Expected:** Only fonts.googleapis.com and fonts.gstatic.com (fonts)
5. **NOT expected:** google-analytics.com, facebook.com, doubleclick.net, or any tracking domains
6. DevTools → Application tab → Cookies → Check "Cookies" section
7. **Expected:** "No cookies" or empty list

**Success Signals:**
- ✅ File:// protocol test: page works without server
- ✅ 4G test: page loads < 2s
- ✅ Lighthouse: all 3 scores ≥ 90 (green)
- ✅ Mobile: responsive, accessible, performant
- ✅ No tracking: no external analytics/tracking domains

**Failure Signals:**
- ❌ File:// test: broken styles, missing content, console errors
- ❌ 4G test: load time > 2s or page weight > 500 KB
- ❌ Lighthouse: any score < 90 (orange or red)
- ❌ Mobile: broken layout, inaccessible touch targets, poor performance
- ❌ Tracking: analytics scripts, tracking pixels, or cookies present
